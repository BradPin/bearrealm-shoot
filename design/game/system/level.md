# 關卡系統（Level）

本文件定義**關卡**在戰役中的編排、多人同步規則、**戰鬥勝利後流程**，以及**跳關**條件與限制。關卡與單場**戰鬥**的細部流程見 `combat.md`；彈珠等級與屬性規則見 `marble.md`。

---

## 關卡與戰役的關係

- **關卡**以**關卡等級**（Lv 1、Lv 2…）編號，對應一場**戰鬥**及其場景、敵方配置與勝利條件。
- 一場**戰役**由連續多個關卡組成；通關一關後依流程進入下一關（關卡等級 +1），直至戰役結束或失敗。

---

## 起始關卡與多人同步

| 規則 | 說明 |
| :--- | :--- |
| **預設起點** | 玩家必須從 **Lv 1** 開始戰鬥（未使用跳關時）。 |
| **多人連線** | 可進行的關卡以**房主**當下能開啟的關卡為準（房主進度／解鎖為權威）。 |

---

## 關卡內容：敵方與行為定義

- 關卡中通常會有**敵人**；敵人**不一定是彈珠**（形式依關卡設計而定）。
- 敵方行為多為**預定義的回合腳本**，依序循環，例如：
  - 回合 1：施放祕法彈幕  
  - 回合 2：施放能量護盾  
  - …之後回到回合 1，以此類推。
- **完整定義方式**（敵人行為、出怪、勝利／失敗條件等）將透過**關卡文件或腳本**承載；實作格式待規格化。

## 戰鬥勝利後流程

戰鬥勝利後，依序進行以下步驟：

1. **彈珠等級提升**  
   本場參與的**四個彈珠**中，若**彈珠等級**低於**本關關卡等級**，則該彈珠等級提升至**該關卡等級**（與 `marble.md`「曾通關之最高關卡難度」之定義一致）。

2. **獎勵挑選（投票）**  
   進入獎勵介面，顯示**三個獎勵**；玩家各自投票想拿的選項。  
   - 若**未全員選同一項**：從中**隨機**決定最終獎勵，但**票數較多的選項有較高機率**被抽中（**非票多必勝**）。  
   - 可理解為：較多人選的獎勵**權重較高**，仍保留隨機性。

3. **遺物編織**  
   進入 **遺物編織**（Relic Weave）畫面；**所有玩家都按下確認**後才進入下一步。

4. **下一關**  
   進入**下一關卡**（**關卡等級 + 1**）。

## 跳關機制

當**房主**使用的彈珠滿足「曾達成之最高通關紀錄」時，可在開戰前選擇**較高的起始關卡**，不必從 Lv 1 開始。

### 解鎖對照（範例）

| 該彈珠曾勝利達成之等級（門檻） | 可選擇之起始關卡（範例） |
| :--- | :--- |
| ≥ **10**（含曾於 Lv 10 取勝） | 至少可從 **Lv 6** 起 |
| ≥ **15** | 至少可從 **Lv 11** 起 |
| ≥ **20** | 至少可從 **Lv 16** 起 |
| … | 每再 **+5** 門檻，起始關卡再 **+5**（見下式） |

**通式**：當該彈珠之通關紀錄 **≥ 10 + 5k**（k = 0, 1, 2, …）時，可將起始關卡設為 **6 + 5k** 或**更低**之合法關卡。

### 跳關時的可選關卡

- 玩家**可以選擇比解鎖上限更低的關卡**（例如已可從 Lv 11 起跳，仍選 Lv 6）。
- **可選關卡編號**必為 **5 的倍數 + 1**，即形如 **5n + 1**（n = 0, 1, 2, …）：  
  **Lv 1、6、11、16、21…**
